4. PROBLEMAS FRECUENTES Y SOLUCIONES
================================================================

PROBLEMA: ModuleNotFoundError: No module named 'django'
SOLUCION: Olvidaste activar el venv.
          Ejecuta primero: venv\Scripts\activate

PROBLEMA: "El modelo aun no esta disponible"
SOLUCION: Falta configurar o descargar el checkpoint.
          a) Edita config\config.yaml y verifica checkpoint_path.
          b) Verifica que la carpeta data\checkpoints\unillanos-v3\
             exista y tenga los archivos .safetensors.
          Si no tienes el checkpoint, descargalo de Colab (ver seccion 7).

PROBLEMA: La pagina no carga en el navegador
SOLUCION: El servidor no esta corriendo. Revisa la terminal.
          Si hay error de Python, copialo y consulta la guia de arquitectura.

PROBLEMA: Puerto 8000 ocupado
SOLUCION: Usa otro puerto:
          python ui\django_chatbot\run.py --port 8080
          Luego abre: 127.0.0.1:8080
          O cierra el proceso que ocupa el puerto (otra instancia del servidor).

PROBLEMA: El modelo tarda mucho en la primera pregunta
SOLUCION: Es normal. El modelo pesa ~3 GB. En CPU (laptop) tarda 1-3 min.
          La interfaz muestra mensajes de progreso mientras espera.
          Las preguntas siguientes son mas rapidas (modelo ya en RAM).

PROBLEMA: El modelo inventa informacion (alucinaciones)
SOLUCION: Es una limitacion conocida del modelo actual (v2/v3).
          Los pares curados en add_knowledge.py atacan los casos mas
          criticos. Si identificas un tema que el modelo aluci na
          sistematicamente, agregar pares para ese tema en add_knowledge.py
          y reentrenar en Colab.

PROBLEMA: El servidor se reinicia solo y el modelo tarda cada vez
SOLUCION: El servidor debe lanzarse con --noreload (ya esta configurado
          en run.py). Si se ejecuta con manage.py runserver sin --noreload,
          Django reinicia al detectar cambios en archivos.

6. REENTRENAR EL MODELO EN COLAB
================================================================

Cuando reentrenar:
  - Se agregaron nuevos PDFs.
  - Se amplio o corrigio el dataset.
  - El test_model.py muestra alucinaciones sistematicas.

Pasos:
  1. Asegurarse de tener el dataset actualizado:
       python scripts\build_dataset.py --mode heuristic
       python scripts\add_knowledge.py
     Verificar: data\dataset\dataset_alpaca.json debe existir y tener
     el numero correcto de pares.

  2. Abrir https://colab.research.google.com
     Subir el notebook: File → Upload Notebook
     Archivo: scripts\colab_train_v3.ipynb

  3. Cambiar a GPU T4:
     Runtime (Entorno de ejecucion) → Change runtime type → T4 GPU → Save

  4. Ejecutar Celda 1 (instalar dependencias, ~1 min)
  5. Ejecutar Celda 2 (subir dataset_alpaca.json con el boton)
  6. Ejecutar Celda 3 (entrenamiento, ~25-35 min)
     NO cerrar el navegador ni minimizar Chrome durante el entrenamiento.
     La barra de progreso muestra el avance y la Training Loss.
     La loss debe bajar de ~2.0 al inicio hasta ~0.3-0.5 al final.

  7. Ejecutar Celda 4 (verificacion antes de descargar)
     Prueba 4 preguntas criticas. Si las respuestas son claramente malas,
     algo salio mal con el dataset. No descargar hasta entender por que.

  8. Ejecutar Celda 5 (descarga, ver seccion 7 para instrucciones detalladas)

  9. Mover archivos descargados a:
       C:\Users\Equipo\OneDrive\Documentos\Tesis\CANUTO\data\checkpoints\unillanos-v3\

  10. Actualizar config\config.yaml:
        checkpoint_path: "data/checkpoints/unillanos-v3"

  11. Reiniciar el servidor y probar:
        python scripts\test_model.py --version v3

7. SCRIPT DE DESCARGA DESDE COLAB (UNO POR UNO)
================================================================

La Celda 5 del notebook ya tiene el script de descarga integrado.
Si por alguna razon necesitas descargarlo manualmente, pega esto en
una celda nueva de Colab:

--------------------------------------------------------------
from google.colab import files
from pathlib import Path

OUTPUT_DIR = "/content/unillanos-v3"   # cambiar si la version es diferente

archivos = sorted([
    f for f in Path(OUTPUT_DIR).iterdir()
    if f.is_file() and not f.name.startswith("_")
])

print(f"{len(archivos)} archivos para descargar.")
print("El navegador abrira un dialogo de descarga por cada archivo.")
print("Acepta CADA descarga antes de presionar Enter.\n")

for i, f in enumerate(archivos, 1):
    size_mb = f.stat().st_size / 1e6
    print(f"[{i}/{len(archivos)}] Descargando: {f.name}  ({size_mb:.0f} MB)")
    files.download(str(f))
    if size_mb > 50:
        input("    Espera a que la barra llegue al 100% y presiona Enter...")
    else:
        print("    (archivo pequeno, continua automaticamente)")

print("\nTodos los archivos descargados.")
print("Muevelos a: CANUTO\\data\\checkpoints\\unillanos-v3\\")
--------------------------------------------------------------

Despues de descargar, mueve todos los archivos a:
  C:\Users\Equipo\OneDrive\Documentos\Tesis\CANUTO\data\checkpoints\unillanos-v3\

Y actualiza config\config.yaml:

  permanece activa por ~4-12 horas despues del entrenamiento).
