# CANUTO — OCR de escaneados con docling — ENTORNO SEPARADO (no mezclar con el venv principal)
#
# docling exige transformers moderno que rompe el stack de inferencia/entrenamiento,
# por eso vive en su propio venv. Se usa solo para OCR-izar los PDFs escaneados.
#
# En la workstation con GPU (Ubuntu, Python 3.12, RTX 5080):
#   python3 -m venv venv-docling && source venv-docling/bin/activate
#   pip install torch --index-url https://download.pytorch.org/whl/cu130
#   pip install -r requirements-docling.txt --extra-index-url https://download.pytorch.org/whl/cu130
# (o simplemente: bash install-docling.sh)
#
# La RTX 5080 es Blackwell y necesita CUDA 13 (cu130). El --extra-index-url evita que,
# al resolver docling, pip cambie torch por una version CPU desde PyPI.
#
# Flujo de dos pasos (MISMO --output-dir):
#   1) venv principal:  python scripts/extract_text.py         (digitales)
#   2) venv-docling:    python scripts/extract_text.py --ocr   (escaneados; salta los ya hechos)
